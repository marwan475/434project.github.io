Optimizations

- Using high bit low bit to store chars in our packet to allow for better usage of the packet space

- instead of using one large database that we would have to maintain, we download database tables from an API for ASNs of a specfic date

- database tables contain IPs and ASNs of those IPs

- If server doesnt have the database table for the requested date it will download it from the API

- Server uses and API to get org name from ASN and ASN from org name

- Server has a cache of databases tables and replaces least recently used table if capacity is reached

- Binary search on database table to find IP/ASN
